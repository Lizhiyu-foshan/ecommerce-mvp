# 图片上传文件类型验证修复说明

## 修复概述
修复了商品图片上传接口缺少文件类型和大小验证的安全问题。

## 修复内容

### 1. 添加文件类型白名单验证
- **允许的文件扩展名**: `.jpg`, `.jpeg`, `.png`, `.gif`, `.webp`
- **验证逻辑**: 提取文件名后缀并转换为小写后进行白名单检查
- **错误响应**: HTTP 400，提示"不支持的文件类型，只允许: .jpg, .jpeg, .png, .gif, .webp"

### 2. 添加文件大小限制验证
- **最大文件大小**: 10MB (10 * 1024 * 1024 bytes)
- **验证逻辑**: 读取文件内容后检查字节数
- **错误响应**: HTTP 400，提示"文件大小超过限制，最大允许: 10.0MB"
- **注意**: 验证后使用 `file.file.seek(0)` 重置文件指针，确保后续处理正常

## 代码变更

### 新增导入
```python
from pathlib import Path
```

### 新增常量
```python
ALLOWED_EXTENSIONS = {".jpg", ".jpeg", ".png", ".gif", ".webp"}
MAX_FILE_SIZE = 10 * 1024 * 1024  # 10MB
```

### 修改函数
在 `upload_product_image` 函数开头添加验证逻辑：
1. 文件类型验证（在检查商品存在之前）
2. 文件大小验证
3. 重置文件指针

## 安全提升
- 防止上传恶意脚本文件（如 .php, .js 等）
- 防止上传超大文件导致服务器资源耗尽
- 符合常规图片上传安全规范

## 测试建议
1. 测试正常图片上传（jpg, png, gif, webp）
2. 测试非法文件类型上传（如 .exe, .php）
3. 测试超大文件上传（>10MB）
4. 测试边界文件大小（刚好10MB）
