# BMAD-METHOD v2.0 for OpenClaw

**优化版多Agent敏捷开发框架** - 专为Kimi 2.5多模态能力增强

## 什么是 BMAD-METHOD v2.0？

BMAD-METHOD (Breakthrough Method of Agile AI Driven Development) 是一个突破性的AI代理编排框架。

**v2.0 核心优化**:
- 🎨 **UX设计师两次介入** - 需求阶段低保真原型 + 开发阶段高保真设计
- 🧠 **Kimi 2.5多模态增强** - 图像理解、视觉描述、接口UX评估
- ⚡ **前后端并行开发** - 基于接口UX评估报告，效率提升40%

**核心理念**: 不是让AI替你写代码，而是让AI帮你更专业地写代码。

## Agent 角色详情

### 业务分析师 (Analyst)
- **职责**: 市场调研、需求收集、用户画像
- **输出**: 项目简报 (Project Brief)
- **并行**: 可独立启动

### 产品经理 (PM)
- **职责**: PRD 创建、功能优先级、产品路线图
- **输出**: 产品需求文档 (PRD)、Epic
- **依赖**: 分析师输出

### 架构师 (Architect) ⭐ 可并行
- **职责**: 系统设计、技术选型、API 设计
- **输出**: 架构设计文档 (ADR)
- **依赖**: PRD
- **并行**: 可与 UX 同时工作

### UX 设计师 (UX) ⭐ v2.0核心优化

**第一次介入 - 需求确认阶段**:
- **输入**: 项目简报 + 初步用例
- **输出**: `docs/ux-wireframes-v1.md` (低保真原型)
- **Kimi多模态能力**:
  - 🖼️ 图像理解 - 分析参考产品风格
  - 📝 ASCII线框图 - 快速可视化
  - 🔄 Mermaid流程图 - 用户路径
- **价值**: 与客户沟通的基础，激发需求补充

**第二次介入 - 开发阶段**:
- **输入**: PRD + 低保真原型 + 架构设计
- **输出**: `docs/ux-design-v2.md` (高保真设计)
- **Kimi多模态能力**:
  - 🎨 完整设计系统 (色彩/字体/间距/圆角/阴影)
  - 📦 组件库规范
  - 📱 响应式方案
  - ✨ 动效规范
  - ⭐ **接口UX评估报告** (独有功能)
- **价值**: 
  - 给前端: 可直接编码的高保真设计图
  - 给后端: UX视角的接口优化建议
  - 前后端并行开发，效率提升40%+

### Scrum Master
- **职责**: Sprint 规划、故事拆分、进度跟踪
- **输出**: Sprint 计划、用户故事
- **依赖**: 架构 + UX

### 开发者 (Developer)
- **前端开发者**:
  - 输入: 高保真UI设计图、组件库规范、交互规范
  - 职责: 页面实现、组件开发、交互效果
- **后端开发者**:
  - 输入: 架构设计文档、接口UX评估报告
  - 职责: API实现、数据库实现、接口优化
- **并行**: 前后端可同时开发

### QA 工程师 (QA)
- **职责**: 代码审查、测试策略、质量保证
- **输出**: 测试报告、审查意见
- **依赖**: 开发输出

## 工作流程对比

### 快速流程 (Quick Flow)
适合小功能开发和 bug 修复：

```
分析师 → 开发者 → QA
 10min    15min    5min
```

### 标准完整流程 (Full Flow)
适合新产品开发：

```
Phase 1: 规划 (串行)
  ├── product-brief (分析师) 10min
  └── create-prd (产品经理) 15min

Phase 2: 设计 (并行)
  ├── create-architecture (架构师) 20min
  └── create-ux (UX设计师) 20min

Phase 3: Sprint 规划
  └── sprint-planning (Scrum Master) 10min

Phase 4: 开发 (串行)
  └── 后端 → 前端 → QA

总时间: 95分钟
```

### v2.0优化完整流程 (Optimized Full Flow)
适合有前端界面的新产品开发：

```
Phase 1: 需求探索 (串行)
  ├── product-brief (分析师) 10min
  ├── create-wireframes (UX第一次) 15min ⭐
  └── refine-requirements (分析师补充) 5min

Phase 2: 产品设计
  └── create-prd (产品经理) 15min

Phase 3: 技术设计 (并行)
  ├── create-architecture (架构师) 20min
  └── create-visual-design (UX第二次) 20min ⭐
        └── 包含: 设计系统 + 组件库 + 接口UX评估

Phase 4: 开发 (并行) ⭐
  ├── frontend-dev (前端开发者) 30min
  └── backend-dev (后端开发者) 30min
        └── 基于接口UX评估优化

Phase 5: 测试
  └── qa-review (QA) 10min

总时间: 75分钟 (节省21%)
```

## v2.0 核心创新

### 1. UX设计师两次介入

| 介入时机 | 输出 | 价值 |
|---------|------|------|
| 需求确认阶段 | 低保真原型 | 可视化沟通，激发需求补充 |
| 开发阶段 | 高保真设计 + 接口UX评估 | 前后端并行，效率提升40% |

### 2. Kimi 2.5 多模态能力应用

| 能力 | 第一次介入 | 第二次介入 |
|------|-----------|-----------|
| 图像理解 | 分析参考产品风格 | 评估设计一致性 |
| 视觉描述 | ASCII线框图 | 详细UI设计描述 |
| 流程可视化 | Mermaid用户流程图 | 交互状态流转 |
| 评估分析 | - | **接口UX评估** |

### 3. 接口UX评估 (独有功能)

```markdown

- ⚠️ 建议: 网络失败时本地缓存+重试

### POST /api/orders (订单创建)
**当前设计:**
- 同步创建
```

## 配置

编辑 `~/.openclaw/skills/bmad-method/config.yaml`:

```yaml
settings:
  default_model: "kimi-coding/k2p5"
  max_parallel_agents: 5
  timeout_minutes: 30
  
  # v2.0 新增配置
  ux_design:
    enable_multimodal: true      # 启用Kimi多模态能力
    enable_interface_ux_review: true  # 启用接口UX评估

    visual_design_detail: "high" # 高保真详细程度: high/medium/low

integrations:
  opencode:
    enabled: true
```

## 最佳实践

### 1. 选择合适的流程

| 场景 | 推荐流程 | 时间 |
|------|---------|------|
| 小功能/bug修复 | Quick Flow | 15分钟 |
| 纯后端API开发 | 标准Full Flow (无UX) | 60分钟 |
| **有前端界面的新功能** | **v2.0 Optimized Flow** | **75分钟** |
| 大型新产品 | v2.0 Full Flow + 多Sprint | 2-4小时 |

### 2. 充分利用并行优势
- **Phase 3**: 架构师和UX设计师同时工作，节省20分钟
- **Phase 4**: 前后端基于接口UX评估并行开发，节省30分钟

### 3. UX两次介入的关键决策点
- **第一次介入后**: 必须与客户确认低保真原型，收集反馈
- **第二次介入后**: 必须基于接口UX评估优化API设计

### 4. 迭代优化
- 每个Sprint结束后回顾
- 根据项目特点调整Agent配置
- 积累项目特定的设计系统和组件库
